# Preview Workflow Rules

Always edit existing HTML/CSS files directly.

Do NOT:
- generate duplicate files
- print huge HTML in chat
- create unnecessary frameworks
- create preview explanations
- create localhost instructions unless requested

Assume:
- user works in VS Code
- Live Server is active
- browser auto-refreshes

Workflow:
Claude edits file
→ user saves
→ browser refreshes automatically

Prioritize:
- direct file editing
- visual iteration speed
- minimal friction
- clean structure
